## 一、全局设计语言

### 1.1 布局基础

- **目标分辨率**：1280×720（桌面），1024×768（平板），响应式缩放
- **宽高比锁定**：16:9，两侧或上下留黑边（letterbox），不拉伸
- **字体**：中文用系统字体栈 `"Noto Serif SC", "Source Han Serif CN", serif`，对话文字用衬线，UI 标签用无衬线
- **颜色基调**：深樱花色 `#2D1B26`（背景底色）+ 暖米白 `#F5ECD7`（文字/对话框）+ 玫红 `#C9506A`（强调色）

### 1.2 动画基调（Framer Motion）

| 动画类型 | duration | easing |
|---------|----------|--------|
| 场景切入 | 0.6s | easeInOut |
| 角色入场 | 0.5s | spring (stiffness 80) |
| 情绪切换 | 0.3s | easeOut |
| 对话框出现 | 0.25s | easeOut |
| 按钮 hover | 0.15s | easeOut |
| 文字打字机 | 每字 30-60ms | linear |

### 1.3 z 轴层级（从后到前）

```
z-0   背景图（全屏铺满）
z-10  背景氛围效果（粒子、光晕，可选）
z-20  角色立绘（居中或偏右）
z-30  对话框 + 文字
z-40  选项按钮
z-50  UI 顶栏（菜单、存档图标）
z-60  全局遮罩（转场、淡入淡出）
z-70  弹出层（存档/设置面板）
z-80  错误提示
```

### Screen 4 — AI 响应状态机 ⭐ 最关键，必须完整实现

**这是产品差异化的核心 UI 路径。每个状态都必须有明确的视觉设计。**

#### State A：LOADING（等待 Claude 响应）

```
┌───────────────────────────────────────────────────┐
│ 凯恒                                              │
│                                                   │
│   ●  ●  ●   （三点波浪动画，0.8s 循环）            │
│                                                   │
│ [输入已禁用，点击无反应]                            │
└───────────────────────────────────────────────────┘
```

- 角色立绘切换到"思考"情绪变体（低头或侧目）
- 顶部进度条极缓慢填充（给人感觉在处理，不是卡住）
- 如果超过 5 秒，显示"凯恒正在认真想你说的话……"

#### State B：STREAMING（流式文字逐字出现）

```
┌───────────────────────────────────────────────────┐
│ 凯恒                                              │
│                                                   │
│ "你说得对，我确实……一直注意着你。只是没想到▋"      │
│         （文字逐字打出，每字 40ms，▋ 是光标）      │
│                                                   │
│ [输入已禁用]                                      │
└───────────────────────────────────────────────────┘
```

- 角色立绘在 streaming 开始时切换情绪（根据响应前几个字判断情感倾向）
- 文字速度可在设置里调整（慢/正常/快）
- 完成后光标消失，输入启用

#### State C：COMPLETE（响应完成，等待玩家操作）

```
┌───────────────────────────────────────────────────┐
│ 凯恒                                              │
│                                                   │
│ "你说得对，我确实一直注意着你。只是没想到你也会     │
│  这样看着我。"                                    │
│                               （点击继续 ▶）      │
└───────────────────────────────────────────────────┘
```

#### State D：ERROR（API 超时或失败）

```
┌───────────────────────────────────────────────────┐
│ 凯恒                                              │
│                                                   │
│ "……"                                             │
│                                                   │
│ （凯恒沉默了一会儿，似乎在整理思绪。）               │
│                                                   │
│ [重试]   [换个话题]                               │
└───────────────────────────────────────────────────┘
```

- **叙事包装**：不显示"网络错误"，而是"凯恒陷入了沉默"，保持沉浸感
- [重试] = 重新调用 API
- [换个话题] = 使用 fallback 脚本台词

#### State E：FALLBACK（使用预写台词替代 AI 响应）

```
┌───────────────────────────────────────────────────┐
│ 凯恒                                              │
│                                                   │
│ "嗯……"  （预写的通用过渡台词）                    │
│                               （点击继续 ▶）      │
└───────────────────────────────────────────────────┘
```

- 状态对玩家透明，不提示"这是备用内容"
- Fallback 台词库：每个场景至少 3 条通用过渡词，随机选取

## 四、前 3 分钟体验 Storyboard

**这是"wow moment"的设计路径。玩家必须在 3 分钟内感受到 AI 对话的独特性。**

```
T+0:00  标题屏加载（背景图 + 音乐淡入）
T+0:10  玩家点击"开始游戏"
T+0:15  角色介绍屏 — 凯恒立绘从右滑入，文字淡入
         "…你来了。"  ← 角色的第一句话，克制，意味深长
T+0:30  玩家点击"开始故事"
T+0:35  场景切换 → 对话主界面（咖啡馆背景，下午阳光）
T+0:40  脚本对话开始（2-3 句引入，纯脚本，不触发 AI）
         凯恒："你今天……看起来有些不同。"
T+1:00  出现第一次选项：
         [A] "只是换了发型。"
         [B] "你观察得很仔细嘛。"
         [C] ✨ 说说你真正想说的（AI 对话）← 醒目提示
T+1:10  如果玩家选 AI 输入框：
         提示文字出现："凯恒在等着听你说…"
         输入框激活，有轻微发光边框
T+1:25  玩家输入第一句话，按发送
T+1:30  角色立绘切换到"思考"情绪，三点动画出现
         ⭐ 关键时刻：玩家第一次意识到"他真的在回应我"
T+1:40  流式文字开始出现 — 凯恒的 AI 生成回应逐字打出
         角色情绪随内容切换（如果是温柔回应 → 微笑立绘）
T+2:00  流式完成。玩家读完，情绪体验：
         "他回应了我说的话，不是预设的，是真的。"
T+2:15  下一轮对话继续，玩家可以再次选择 AI 输入或预设
T+3:00  第一章第一场景的核心情感节点：
         凯恒说了一句让玩家想截图/分享的台词
         （需要在剧情脚本里特别设计这个"钩子"）
```

**设计原则：** 第一次 AI 响应必须：
1. 速度 < 4 秒（用较短的 prompt 确保首次体验快速）
2. 内容紧扣玩家输入（Claude 的上下文理解在这里体现）
3. 角色情绪变化可见（立绘切换同步）

      "next": "choice_001"
    },
    {
      "type": "choice",
      "id": "choice_001",

        },

    },
