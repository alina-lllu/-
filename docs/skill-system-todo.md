# Skill 系统设计 TODO

## 结论

当前项目不是传统重度 SLG，更适合做「AI 乙女 VN + 轻策略养成」。

Skill 不建议先做战斗技能，而应围绕三件事设计：

1. 影响对话：让玩家在关键对话中获得不同表达方式。
2. 影响关系：让角色好感、信任、记忆解锁出现差异。
3. 影响剧情：让部分分支、事件、结局由长期养成结果触发。

## Skill 类型

| 类型 | 用途 | 示例 | MVP 优先级 |
|---|---|---|---|
| 对话技能 | 改变玩家在对话节点的可选策略 | 观察、共情、追问、沉默、直球 | P0 |
| 情绪技能 | 影响 AI 角色回应的情绪倾向 | 安抚、试探、打断、靠近、转移话题 | P0 |
| 关系技能 | 改变好感度、信任度、距离感等关系数值 | 记住细节、保持边界、主动关心 | P1 |
| 记忆技能 | 解锁角色过去、隐藏台词、特殊回忆 | 细节回想、旧事触发、共同记忆 | P1 |
| 探索技能 | 在场景中发现额外信息或隐藏选项 | 观察环境、阅读气氛、发现物件 | P2 |
| 资源技能 | 如果后续 SLG 化，用于管理时间、体力、行动点 | 时间管理、行动规划、专注力 | P2 |
| 角色路线技能 | 影响进入不同男主路线或隐藏路线 | 建筑审美、文学话题、社交洞察 | P3 |

## MVP 推荐技能池

第一版只做 5 个技能，避免系统过重。

| Skill | 玩家感知 | 系统效果 | 适合凯恒路线的原因 |
|---|---|---|---|
| 观察 | 看见对方没说出口的细节 | 解锁额外对话选项；AI prompt 增加「玩家注意到细节」上下文 | 凯恒本身是细节型角色，匹配人设 |
| 共情 | 接住角色情绪，但不逼问 | 正向情绪回应概率提高；好感度小幅增加 | 适合慢热关系 |
| 追问 | 推进更深层信息 | 可能解锁隐藏记忆；也可能触发 distant 情绪 | 有风险收益，适合分支 |
| 边界感 | 不强迫角色表达 | 降低负面回应；提高信任度 | 凯恒怕靠近，边界感很关键 |
| 直球 | 明确表达玩家态度 | 好感度波动更大；触发 caught / slight_smile | 制造高情绪节点 |

## 剧情脚本 TODO

1. 在 `choice` 节点支持 skill 条件选项
   - 例：`requiredSkill: { id: "observe", level: 1 }`
   - 未满足时隐藏，不显示灰色禁用，避免破坏沉浸感。

2. 在 `ai_dialogue` 节点支持 skill 上下文
   - 玩家使用「共情」时，system prompt 追加：玩家选择了温和承接情绪。
   - 玩家使用「追问」时，system prompt 追加：玩家正在靠近角色的回避点。

3. 在章节结尾支持 skill 结算
   - 展示本章最常使用的技能倾向。
   - 根据技能使用给少量经验。

4. 给第一章补 3 个技能节点
   - scene01：第一次 AI 输入前，引导「观察 / 直球」。
   - scene02：雨天停留时，引导「共情 / 追问 / 边界感」。
   - scene03：分别前，引导「直球 / 边界感」影响结尾。

## AI Prompt TODO

1. 给每个 skill 定义 prompt 片段
   - `observe`：玩家注意到细节，但不要让凯恒立刻解释太多。
   - `empathy`：玩家温和承接情绪，凯恒可以稍微放松。
   - `probe`：玩家追问敏感点，凯恒可能回避或短暂沉默。
   - `boundary`：玩家保持距离，凯恒会觉得安全。
   - `direct`：玩家直接表达，凯恒会被打乱节奏。

2. 控制 prompt 长度
   - 每次只注入当前使用技能，不把所有技能说明塞进上下文。
   - 保持 AI 首次响应 < 4 秒的体验目标。

3. 建立 skill 到 emotion 的弱映射
   - `observe` → `attentive`
   - `empathy` → `warm`
   - `probe` → `distant` 或 `caught`
   - `boundary` → `neutral` 或 `warm`
   - `direct` → `caught` 或 `slight_smile`

## 数值规则 TODO

1. 技能等级只做 1-3 级
   - Level 1：解锁基础技能选项。
   - Level 2：提高正向回应概率或解锁一条隐藏台词。
   - Level 3：影响章节结尾或角色记忆。

2. 技能经验来源
   - 选择对应技能选项。
   - AI 对话被判定为同类倾向。
   - 章节结算给主倾向技能额外经验。

3. 不做复杂资源消耗
   - MVP 不消耗体力、行动点、金币。
   - 后续如果改成真正 SLG，再加时间 / 行动力系统。

## 第一版验收标准

1. 第一章至少有 3 个技能选择节点。
2. 至少 5 个技能可被记录和升级。
3. 技能选择会影响 AI 回复 prompt。
4. 技能选择会影响好感度或情绪，但不直接暴露数值。
5. 存档后重新进入，技能状态不丢失。
6. 章节结尾能显示玩家本章互动倾向。
7. 没有 API Key 时，技能节点仍可通过 fallback 台词跑通。

## 暂不做

1. 不做战斗技能。
2. 不做复杂技能树。
3. 不做抽卡式技能获取。
4. 不做装备、道具、属性克制。
5. 不做多人角色之间的技能平衡。
6. 不做重 SLG 的资源生产、建筑升级、兵种系统。
